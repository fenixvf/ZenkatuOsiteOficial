# Zenkatu Streaming

Site de streaming independente focado em animes, com painel admin, autenticação Google via Firebase, sistema de comentários e player flexível.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/zenkatu run dev` — run the frontend (port 24662)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + Tailwind CSS + Framer Motion + wouter
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Auth: Firebase Authentication (Google OAuth)
- Realtime data: Firestore (comments)
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Video: HLS.js for m3u8 streams

## Where things live

- `lib/api-spec/openapi.yaml` — API contract (source of truth)
- `lib/db/src/schema/` — Drizzle table definitions (users, obras, episodios, comentarios)
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/zenkatu/src/` — React frontend
- `artifacts/zenkatu/src/lib/firebase.ts` — Firebase config
- `artifacts/zenkatu/src/lib/auth-context.tsx` — Auth context + useAuth hook
- `artifacts/zenkatu/src/pages/` — All app pages
- `dist/avatars/` — Uploaded user avatar files (WebP)

## Architecture decisions

- Firebase Auth for Google OAuth; user data mirrored to PostgreSQL via `/api/usuarios` upsert on every login
- Avatars stored as WebP files on disk (converted from any format client-side via Canvas API before upload)
- Player field (`playerContent`) accepts URL, `<iframe>`, or full HTML embed — detected at render time
- Comments stored in PostgreSQL (not Firestore); Formspree used only for email notifications
- Query params removed from GET endpoints to avoid Orval TS2308 name collision; filtering done client-side
- `lib/api-zod/src/index.ts` exports only `./generated/api` (not `./generated/types`) to prevent Params name collision

## Gotchas

- After any OpenAPI spec change: run `pnpm --filter @workspace/api-spec run codegen` then rebuild libs with `pnpm run typecheck:libs`
- Do NOT add query parameters to GET endpoints — they generate `*Params` types that collide between `generated/api.ts` and `generated/types/`
- VITE_ prefixed env vars are needed for frontend Firebase access (they mirror the secret values)
- The `lib/api-zod/src/index.ts` barrel is regenerated by Orval on every codegen run
