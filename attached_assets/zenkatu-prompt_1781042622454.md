### Player de Vídeo
- Área principal de player: renderiza o conteúdo de player de forma flexível:
  - Se for **URL direta** (mp4, m3u8): usar `<video>` ou HLS.js

- Dropdown de temporada acima da grade de episódios

## 👤 PERFIL E CUSTOMIZAÇÃO (`/perfil`)

### Página de Perfil
- Foto de perfil circular grande, botão de editar com ícone de câmera sobreposto
- Username editável (campo de texto com validação: 3–20 chars, apenas letras/números/underscore)
- Email (somente leitura, vindo do Firebase)
- Botão "Salvar alterações" com feedback visual (loading → sucesso)
- Histórico de assistidos (cards compactos)
- Minha lista / Favoritos

### Upload de Foto de Perfil
- O usuário seleciona uma imagem via input file
- **Conversão automática para WebP** no frontend antes do envio (usando Canvas API):
  ```js
  // Converter para WebP com qualidade 0.75 antes de enviar
  canvas.toBlob(blob => upload(blob), 'image/webp', 0.75)
  ```
- Enviar a imagem convertida para o servidor (API Route Next.js `/api/upload-avatar`)
- O servidor salva o arquivo em storage externo (ou diretamente no PostgreSQL como `bytea` ou como URL de arquivo)
- **Lógica de exclusão:** antes de salvar a nova foto, a API verifica se existe uma `photoURL` antiga no banco para aquele `uid` e a exclui do storage/banco antes de inserir a nova
- Salvar a URL final da foto no PostgreSQL e no Firestore (`users` collection)

### Schema PostgreSQL para Perfis
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  uid VARCHAR(128) UNIQUE NOT NULL,       -- UID do Firebase
  email VARCHAR(255) UNIQUE NOT NULL,
  username VARCHAR(20) UNIQUE,
  photo_url TEXT,                          -- URL da foto atual (WebP)
  role VARCHAR(20) DEFAULT 'user',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

## 🎨 DESIGN SYSTEM DETALHADO

### Componentes Base
- `<Card>` — bordas arredondadas `rounded-xl`, fundo `#0F1C2E`, hover: `border border-blue-500/30 shadow-blue-500/10`
- `<Button primary>` — fundo `#2563EB`, hover: `#1D4ED8`, transição 200ms
- `<Button ghost>` — borda `blue-500/40`, hover: fundo `blue-500/10`
- `<Badge>` — pill pequeno com cor por categoria
- `<Input>` — fundo `#0A1628`, borda `#1E3A8A`, focus: `ring-2 ring-blue-500`
- `<Modal>` — backdrop blur escuro, animação scale-in com Framer Motion
- `<Toast>` — canto inferior direito, auto-dismiss 3s

### Animações (Framer Motion)
- Page transitions: `opacity 0→1` + `y: 20→0`
- Card hover: `scale: 1.03` suave
- Sidebar: `x: -100%→0` com spring
- Banner carousel: `AnimatePresence` com slide
- Skeleton loaders em todas as listas enquanto carrega dados

### Responsividade
- Mobile-first com breakpoints Tailwind: `sm (640px)`, `md (768px)`, `lg (1024px)`, `xl (1280px)`
- Sidebar vira bottom-nav no mobile
- Grid de cards: 1 col mobile → 2 col tablet → 4 col desktop
